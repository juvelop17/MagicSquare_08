### TP-01 — 정상: 완성 성공(시도 1 또는 시도 2 중 먼저 성공한 규칙 반영)

| 항목 | 내용 |
|------|------|
| **테스트 ID** | TP-01 |
| **테스트 일자** | _(실행 일)_ |
| **테스트 목적** | 유효 입력(INV 만족)에 대해 FR-02·FR-03·FR-05를 거쳐 **성공 시 `int[6]`** 이 계약(BR-07·BR-08)과 일치함을 검증한다. |
| **테스트 기능** | FR-02 빈칸 탐색, FR-03 누락 숫자, FR-04·FR-05 두 고정 시도 및 결과 조립 |
| **추적** | AC-06, AC-08, AC-12 (및 시도 2만 성공 시 AC-13 성격) · 시나리오 표 §9.1 TP-01 |

**입력값**

- PRD §9.3 **시드 A** — `int[][]` (행 × 열, 0 = 빈칸):

| 열1 | 열2 | 열3 | 열4 |
|-----|-----|-----|-----|
| 16 | 2 | 3 | 13 |
| 5 | 11 | 10 | 8 |
| 9 | 7 | 0 | 12 |
| 4 | 14 | 15 | 0 |

**테스트 단계**

| 순서 | 조작·관찰 |
|------|-----------|
| 1 | 입력 행렬을 완성 API(또는 Boundary 진입점)에 전달한다. |
| 2 | 빈칸이 row-major 순으로 `(3,3)`, `(4,4)` (1-index)임을 도메인 결과 또는 로그로 확인한다. |
| 3 | 누락 숫자가 `{1, 6}` (`n_small=1`, `n_large=6`)임을 단언한다. |
| 4 | 시도 1·2 순서(FR-05)에 따라 최초 성공 배치의 `int[6]=[r1,c1,n1,r2,c2,n2]` 를 얻는다. |
| 5 | §9.4 헬퍼로 완성 그리드의 행·열·두 대각선 합이 34인지, 1~16 사용이 한 번씩인지 확인한다. |
| 6 | 반환 `int[6]` 이 리포지토리 단위 테스트에 고정된 기대 배열과 **바이트 단위 동일**함을 확인한다 (NFR-03). |

**테스트 환경**

| 항목 | 내용 |
|------|------|
| HW/OS | 개발 PC (Windows 10 이상 등) |
| 런타임 | Python 3.10+, 프로젝트 `.venv` 활성화 |
| 도구 | `python -m pytest` |

**전제 조건**

- FR-01 검증을 통과하는 입력만 경로에 진입한다.
- 호출자 행렬은 호출 전후 동일 원소를 유지한다(NFR-04).

**성공 / 실패 기준**

| 결과 | 조건 |
|------|------|
| 성공 | 오류 없이 `int[6]` 반환, §9.4 재검증 통과, 고정 기대값과 일치 |
| 실패 | 예외·잘못된 코드, 또는 기대 `int[6]` 과 불일치, 또는 입력 배열이 변조됨 |

**특별하게 필요한 절차**

- 기대 `int[6]` 은 PRD §9.3의 “구현 확정 후 단위 테스트 상수”와 동일해야 한다. 시드 A에 대해 수동 검증 시 시도 2 성공 시 `[3, 3, 6, 4, 4, 1]` 형태가 되는지 확인할 수 있다(구현과 테스트 상수가 단일 출처여야 함).

### TP-02 — 정상: 시도 1 실패 후 시도 2 성공

| 항목 | 내용 |
|------|------|
| **테스트 ID** | TP-02 |
| **테스트 일자** | _(실행 일)_ |
| **테스트 목적** | 시도 1이 마방진이 아니고 시도 2만 성공할 때 **출력이 역순 규칙** (`n1=n_large`, `n2=n_small`) 을 따르는지 검증한다. |
| **테스트 기능** | FR-05 두 번째 시도 경로, BR-08 |
| **추적** | AC-13 · §9.1 TP-02 · 시드 B |

**입력값**

- PRD §9.3 **시드 B** — 저장소 내 테스트 픽스처 또는 인접 주석에 명시된 `int[][]` (유효 입력, 시도 1 실패·시도 2 성공 전용).

**테스트 단계**

| 순서 | 조작·관찰 |
|------|-----------|
| 1 | 시드 B를 완성 API에 입력한다. |
| 2 | 내부적으로 시도 1 완성 그리드가 마방진이 **아님**을 확인한다(테스트 더블 또는 헬퍼). |
| 3 | 시도 2 완성 후 마방진이 됨을 확인한다. |
| 4 | 반환 `int[6]` 이 `n1=n_large`, `n2=n_small` 규칙을 만족하고 픽스처에 고정된 기대 배열과 일치한다. |
| 5 | §9.4 속성 검증을 통과한다. |

**테스트 환경** | TP-01과 동일 |
**전제 조건** | 시드 B 정의가 테스트 코드와 문서에서 동일해야 한다. |
**성공 / 실패 기준** | 성공 시 역순 규칙 반영된 고정 `int[6]` · 실패 시 잘못된 순서 또는 `E_DOMAIN` 오판정 |
**특별하게 필요한 절차** | 시드 B 행렬과 기대 `int[6]` 을 `tests/` 픽스처 파일로 버전 관리한다. |

## 부록: 오류 코드·메시지 일람 (경계)

| code | message |
|------|---------|
| `E_SIZE` | `Grid must be 4 by 4.` |
| `E_EMPTY_COUNT` | `There must be exactly two empty cells (value 0).` |
| `E_VALUE_RANGE` | `Each cell must be 0 or an integer from 1 to 16.` |
| `E_DUPLICATE` | `Values from 1 to 16 must not repeat except for 0.` |
| `E_DOMAIN` | `No valid magic square completion exists for this grid.` |
