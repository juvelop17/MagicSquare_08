|------|------|
| 목적 | RED 단계에서 추가할 **pytest 스펙의 체크리스트** (계약·추적용) |

---

## 1. 진행 순서 (RED → GREEN → REFACTOR)

1. **RED (Dual-Track 병렬):** 아래 Track A·Track B ID 단위로 실패하는 테스트를 추가한다 (PRD §8.3).
2. **GREEN:** 각 트랙에서 최소 구현으로 통과.
3. **REFACTOR:** 동작 유지·중복 제거·ECB 방향 확인.

금지: 도메인만 전부 구현한 뒤 경계 검증을 나중에 추가 (PRD §8.3).

---

## 2. Track A — Boundary 계약 (PRD §8.1)

각 케이스는 **`code` + `message` 문자 단위 일치**를 단언한다 (PRD §5 FR-01, §8.1).

| ID | 초점 | 입력 요약 | 기대 `code` | 기대 `message` (전문) |
|----|------|-----------|-------------|-------------------------|
| U-01 | 크기 | 행 또는 열 길이 ≠ 4 (예: 3×4, 4×3) | `E_SIZE` | `Grid must be 4 by 4.` |
| U-02 | 빈칸 개수 | `0`인 칸 0개 | `E_EMPTY_COUNT` | `There must be exactly two empty cells (value 0).` |
| U-03 | 빈칸 개수 | `0`인 칸 3개 이상 | `E_EMPTY_COUNT` | 동일 |
| U-04 | 값 범위 | `0`도 아니고 `1`~`16`도 아닌 셀 (예: 17, -1) | `E_VALUE_RANGE` | `Each cell must be 0 or an integer from 1 to 16.` |
| U-05 | 중복 | `0` 제외 동일 값 2회 이상 | `E_DUPLICATE` | `Values from 1 to 16 must not repeat except for 0.` |
| U-06 | 성공 전달 | 도메인(Mock)이 고정 `int[6]` 반환 | 경계가 **동일** `int[6]` 전달 (직렬화/포맷 일치) | — |
| U-07 | 도메인 불가 | 유효 입력인데 완성 불가(도메인/Mock) | `E_DOMAIN` | `No valid magic square completion exists for this grid.` |

**AC-05 (도메인 미호출):** U-01~U-05에 대해 스키마 검증 실패 시 **도메인의 마방진·해 탐색 로직이 호출되지 않음**을 테스트가 단언한다 (PRD FR-01).

## 3. Track B — Domain 로직 (PRD §8.2)

전제: 단위 테스트에서 도메인 API를 호출할 때는 **INV-01~04를 만족하는 입력**을 쓰거나, INV 위반 경로는 “도메인 단독” 실패 규칙을 문서·테스트로 고정한다 (PRD FR-02 노트, Report/2 §1.4).

| ID | 초점 | 입력·조건 요약 | 기대 |
|----|------|------------------|------|
| D-01 | 성공 경로·출력 형식 | 시드 A (§5) — 유효 그리드 | `[r1,c1,n1,r2,c2,n2]` 가 FR-05·BR-08과 일치 (1-index) |
| D-02 | 빈칸 순서 | 시드 A 또는 시드 B | 첫/둘째 빈칸이 row-major 순서와 일치 (AC-06) |
| D-03 | 시도 1 성공 분기 | 시도 1만 마방진이 되는 구성 | `n1=n_small`, `n2=n_large` |
| D-04 | 시도 2 성공 분기 | 시도 1 실패 후 시도 2 성공 (시드 B) | `n1=n_large`, `n2=n_small` (AC-13) |
| D-05 | 마방진 참 | 알려진 완성 4×4 마방진 | `isMagicSquareComplete` → 참 (AC-10) |
| D-06 | 마방진 거짓 | 완전 채움·합 조건 깨진 그리드 | 거짓 (AC-11) |
| D-07 | 빈칸 INV 위반 | 도메인 단독 호출 시 0개·1개·3개 등 | 설계한 실패 표현과 일치 (Report/2 D-07) |
| D-08 | 중복·경계 중복 | INV-03 위반 입력 | 경계 선행 시 도메인 미호출과 정합 |

**기대 `int[6]` (시드 A):** PRD §9.3은 행렬·첫/둘째 빈칸·누락 `{1,6}` 까지 명시하고, **수치 기대 배열은 단위 테스트 상수로 고정**한다고 한다. RED 문서에서는 FR-05 규칙에 따라 GREEN 단계에서 계산·검증 후 픽스처에 고정한다.

## 4. 통합 시나리오 — TP (PRD §9.1)

| ID | 유형 | 기대 |
|----|------|------|
| TP-01 | 정상: 시도 1 성공 | `int[6]` 고정 기대 |
| TP-02 | 정상: 시도 1 실패 → 시도 2 성공 | 역순 규칙 반영 |
| TP-03 | 실패: 두 시도 모두 실패 | `E_DOMAIN` + 고정 메시지 |
| TP-04 | 입력 오류: 크기 | `E_SIZE` |
| TP-05 | 입력 오류: 빈칸 개수 | `E_EMPTY_COUNT` |
| TP-06 | 입력 오류: 값 범위 | `E_VALUE_RANGE` |
| TP-07 | 입력 오류: 중복 | `E_DUPLICATE` |

## 6. 테스트 데이터 — 시드 행렬 (PRD §9.3)

### 시드 A (시도 1 성공 경로 검증용 소재로 명시)

| 열1 | 열2 | 열3 | 열4 |
|-----|-----|-----|-----|
| 16 | 2 | 3 | 13 |
| 5 | 11 | 10 | 8 |
| 9 | 7 | 0 | 12 |
| 4 | 14 | 15 | 0 |

- 1-index 첫 빈칸 `(3,3)`, 둘째 `(4,4)`.
- 누락 숫자: `1`, `6` (`n_small=1`, `n_large=6`).

### 시드 B (시도 1 실패 · 시도 2 성공)

- 행렬은 테스트 모듈 주석 또는 픽스처 파일에 명시하고, 기대 `int[6]`을 함께 고정 (PRD §9.3).

### 시드 C (두 시도 모두 실패)

- INV를 만족하면서 두 배치 모두 비마방진인 예 1건 이상 (PRD §9.3).

### 속성 재검증 (PRD §9.4)

- 성공 시 완성 그리드에 대해 행·열·두 대각선 합 34, 1~16 한 번씩 등은 헬퍼로 재검증 가능.
