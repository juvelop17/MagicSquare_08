## 2. 가상환경 생성 (최초 1회)

프로젝트 루트에서:

```bash
python -m venv .venv
```

---

## 3. 가상환경 활성화 (매 작업 세션)

### 3.1 Windows — PowerShell (권장)

```powershell
cd c:\Users\usejen_id\dev\MagicSquare_08
.\.venv\Scripts\Activate.ps1
```

- 프롬프트 앞에 `(.venv)` 가 보이면 활성화된 상태다.

**실행 정책 오류가 날 때** (현재 사용자 범위에서만 허용 예시):

```powershell
Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser
```

### 3.2 Windows — 명령 프롬프트 (cmd)

```bat
cd c:\Users\usejen_id\dev\MagicSquare_08
.venv\Scripts\activate.bat
```

### 3.3 macOS / Linux (bash 등)

```bash
cd /path/to/MagicSquare_08
source .venv/bin/activate
```

---

## 4. 의존성 설치 (가상환경 활성화 후)

### 4.1 pip 업그레이드

```bash
python -m pip install --upgrade pip
```

### 4.2 개발 의존성 포함 편집 가능 설치

`pyproject.toml` 의 `[project.optional-dependencies] dev` 에 맞춰 `pytest`·`pytest-cov` 등을 함께 설치한다:

```bash
pip install -e ".[dev]"
```

- 패키지 `magic_square` 가 편집 가능 모드로 설치되어 `import magic_square` 및 테스트 수집이 일관되게 동작한다.

**최소만 설치**해도 되는 경우 (예: `pytest`만):

```bash
pip install pytest
```

루트 `pyproject.toml` 의 `[tool.pytest.ini_options]` 에 `pythonpath = ["."]` 가 있어 소스 트리 인식에 유리하다.

---

## 5. 테스트 실행

가상환경이 **켜진 상태**에서 프로젝트 루트에서:

| 목적 | 명령 |
|------|------|
| 전체 테스트 | `python -m pytest` |
| TP 시나리오만 (간단 출력) | `python -m pytest tests/test_tp_scenarios.py -q` |
| 커버리지 (예시) | `python -m pytest --cov=magic_square --cov-report=term-missing` |

테스트 경로·옵션 기본값은 `pyproject.toml` 의 `[tool.pytest.ini_options]` 를 따른다.

---

## 6. 작업 종료 시

가상환경 비활성화:

```bash
deactivate
```

---

## 7. IDE / Cursor

Python 인터프리터를 **`.venv\Scripts\python.exe`** (Windows) 또는 **`.venv/bin/python`** (Unix) 로 지정하면 터미널과 동일한 환경에서 테스트·디버깅이 맞는다.
