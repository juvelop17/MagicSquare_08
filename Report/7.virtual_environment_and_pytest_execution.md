## 3. 가상환경 활성화 (매 작업 세션)

### 3.1 Windows — PowerShell (권장)

```powershell
cd c:\Users\usejen_id\dev\MagicSquare_08
.\.venv\Scripts\Activate.ps1
```

- 프롬프트 앞에 `(.venv)` 가 보이면 활성화된 상태다.

**실행 정책 오류가 날 때** (현재 사용자 범위에서만 허용 예시):

```powershell
Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser
```

### 3.2 Windows — 명령 프롬프트 (cmd)

```bat
cd c:\Users\usejen_id\dev\MagicSquare_08
.venv\Scripts\activate.bat
```

### 3.3 macOS / Linux (bash 등)

```bash
cd /path/to/MagicSquare_08
source .venv/bin/activate
```

## 4. 의존성 설치 (가상환경 활성화 후)

### 4.1 pip 업그레이드

```bash
python -m pip install --upgrade pip
```

### 4.2 개발 의존성 포함 편집 가능 설치

`pyproject.toml` 의 `[project.optional-dependencies] dev` 에 맞춰 `pytest`·`pytest-cov` 등을 함께 설치한다:

```bash
pip install -e ".[dev]"
```

- 패키지 `magic_square` 가 편집 가능 모드로 설치되어 `import magic_square` 및 테스트 수집이 일관되게 동작한다.

**최소만 설치**해도 되는 경우 (예: `pytest`만):

```bash
pip install pytest
```

루트 `pyproject.toml` 의 `[tool.pytest.ini_options]` 에 `pythonpath = ["."]` 가 있어 소스 트리 인식에 유리하다.

---

## 5. 테스트 실행

가상환경이 **켜진 상태**에서 프로젝트 **루트** (`MagicSquare_08`) 에서 실행한다.

### 5.1 기본 명령

| 목적 | 명령 |
|------|------|
| `tests/` 아래 전체 | `python -m pytest` |
| 특정 파일만 | `python -m pytest tests/test_tp_scenarios.py` |
| 간단 출력 (`-q`) | `python -m pytest tests/test_tp_scenarios.py -q` |

### 5.2 `pyproject.toml` 과의 관계

| 설정 | 의미 |
|------|------|
| `testpaths = ["tests"]` | 인자 없이 돌릴 때 `tests/` 만 수집 |
| `pythonpath = ["."]` | 루트를 경로에 두어 `import magic_square` 등이 안정적 |
| `addopts = "-v"` | 기본으로 테스트 이름·결과가 한 줄씩 자세히 출력 |

커버리지는 **별도 플래그**(`--cov`, 아래 §6)를 줄 때만 수행된다.

## 6. 테스트 커버리지 확인 (`pytest-cov`)

### 6.1 전제

- `[project.optional-dependencies] dev` 에 `pytest-cov` 가 포함되어 있다.
- **권장:** `pip install -e ".[dev]"` 로 한 번에 설치한다.
- `pytest` 만 설치한 상태에서는 `pip install pytest-cov` 로 보완한다.

### 6.2 터미널 요약 + 미커버 줄 표시

대상 패키지는 프로젝트 소스 트리 이름인 **`magic_square`** 이다.

```bash
python -m pytest --cov=magic_square --cov-report=term-missing
```

- 터미널에 **파일별 커버리지 %** 와 **`Missing` 열(실행되지 않은 줄)** 이 함께 나온다.

### 6.3 HTML 리포트

브라우저에서 줄 단위로 보기 좋게 하려면:

```bash
python -m pytest --cov=magic_square --cov-report=html --cov-report=term-missing
```

- 생성 디렉터리: 루트 기준 **`htmlcov/`** → **`htmlcov/index.html`** 을 연다.
- `.gitignore` 에 `htmlcov/` 를 두면 리포트가 저장소에 올라가지 않게 할 수 있다.

### 6.4 README·PRD 와의 연결

- 루트 [`README.md`](../README.md) 에도 동일 계열 예시가 있다.
- PRD·`.cursorrules` 에서 말하는 **커버리지 하한**(예: 80%)은 위 명령으로 로컬에서 확인한 뒤, 필요 시 CI 게이트와 맞춘다.
