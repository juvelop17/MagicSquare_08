## 2. RED 선택 내역과 이유

### 2.1 선택한 RED 테스트

- **L-RED-01**: `find_blank_coords()` 가 row-major 순서로 빈칸 좌표 2개를 반환
- **U-RED-01**: 입력이 4x4가 아니면 Boundary `validate()` 가 예외 발생

### 2.2 선택 이유 (요약)

1. Domain 규칙(row-major)과 Boundary 계약(4x4)을 동시에 고정해 Dual-Track 시작점으로 적합했다.  
2. 한 커밋 단위에서 GREEN 최소 구현까지 닫을 수 있는 작은 범위였다.

---

## 3. GREEN 최소 구현 결과

### 3.1 Track B (Logic / Domain)

- `magicsquare/domain.py`
  - `find_blank_coords(grid)` 추가
  - `MATRIX_SIZE` / `EMPTY_VALUE` 기반 row-major 스캔
  - 반환 형식: `((r1,c1),(r2,c2))` (1-index)

### 3.2 Track A (UI / Boundary)

- `magicsquare/boundary.py`
  - `validate(grid)` : 4x4 형상 검증, 실패 시 `ValueError`
  - `solve(grid)` : 기존 도메인 흐름(`magic_square.boundary.api.complete_magic_square`) 위임 호출
  - 실패 응답(`CompletionError`)은 `ValueError(message)`로 변환

### 3.3 공통 상수

- `magicsquare/constants.py`
  - `MATRIX_SIZE = 4`
  - `EMPTY_VALUE = 0`

## 4. GUI MVP 구현 (Screen 전용)

### 4.1 추가 구조

- `magicsquare/gui/__init__.py`
- `magicsquare/gui/app.py`
- `magicsquare/gui/__main__.py`

### 4.2 MVP 동작

1. 4x4 `QSpinBox` 입력 그리드(0~16)  
2. `풀기` 버튼 클릭 시 `validate -> solve` 순서 호출  
3. 오류 시 `QMessageBox`로 메시지 표시  
4. 성공 시 결과 `int[6]`를 라벨에 표시

### 4.3 의존성/진입점

- `pyproject.toml`
  - optional dependency: `gui = ["PyQt6>=6.7"]`
  - 패키지 include에 `magicsquare*` 추가
- `README.md`
  - 공식 실행 경로 1개 문서화: `python -m magicsquare.gui`
