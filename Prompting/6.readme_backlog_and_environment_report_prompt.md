### 1. **작업 단위·우선순위의 본체 — `docs/PRD_MagicSquare_4x4_TDD.md`**
- **기능 요구사항(FR)**, 범위(In/Out), 입출력 계약, 테스트 ID 추적은 여기가 기준입니다.

### 2. **에픽·시나리오로 나누기 — `Report/4.user_journey_epic_to_technical_scenario.md`**
- PRD 표에서도 **골격·Epic·여정** 정렬로 이쪽을 가리킵니다.
- 큰 덩어리(에픽) → 시나리오 → 세부 할 일 로 내릴 때 참조하면 됩니다.

### 3. **Dual-Track·계약·레이어링 — `Report/2.4x4_magic_square_dual_track_tdd_design.md`**
- 경계 트랙 vs 도메인 트랙, 계약·Invariant 정리가 필요할 때 To-Do를 **트랙별**로 나누는 데 유용합니다.

### 4. **문제 정의·용어 맞추기 — `Report/1.4x4_magic_square_problem_recognition.md`**
- “무엇을 풀고 있는지”를 다시 맞출 때만 필요하면 됩니다. 할 일 목록의 **세부 태스크**보다는 배경·용어 통일용입니다.

### 5. **품질·규칙 체크리스트 — `Report/3.cursorrules_specification_and_ai_guidelines.md` + 루트 `.cursorrules`**
- PRD에도 **품질·개발 원칙**으로 묶여 있습니다.
- To-Do 항목에 “타입 힌트·커버리지·ECB·TDD 순서” 같은 **완료 조건**을 붙일 때 참조합니다.

### 6. **문서 작업만 할 때 — `Report/5.prd_authoring_and_document_export.md`**
